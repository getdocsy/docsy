Introduction

If you are offering a technical product, the products documentation plays an important part in the buying decisions of your customers. The State of Self-Service Content Experiences1 found that 99% of users (n=186) said technical content is a primary or secondary criteria in the decision to purchase.

This analysis focuses on the experience of a visitor, that hasn't used your product yet.

Methodology

We make the following assumptions backed by the data of a 2024 Technical Content Report2, based on nearly 100 million sessions:

  • The average visitor spends 5 minutes on the documentation.
  • The average visitor opens 5 pages during their visit.
  • On average, 15% of sessions include in-site-search.
  • Up to 60% of traffic for public documentation comes from external searches.

We are using the terminology of the Divio Documentation System3 to classify content types by their functions: tutorials, how-to guides, technical reference and explanation. Each of those content types requires a different structure and style to be effective.

What we check: We check if each page has a structure that matches its function. We do this by couting the number of subheadings of each type: tutorial, how-to guide, reference and explanation.

Why it's important: Because users can land on any page, every page needs to be easy to understand. If a single page serves multiple functions, like a how-to guide that also contains reference information and a long design explanation, it is confusing for the user. We chose to check the subheadings because they allow a user to quickly understand what a page is about.
